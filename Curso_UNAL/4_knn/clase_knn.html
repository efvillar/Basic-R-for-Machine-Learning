Aplicacin de la tcnica de los vecinos ms cercanos

Antes de aplicar la tcnica de los vecinos ms cercanos se particiona el conjunto de datos en entrenamiento y validacin:

set.seed(1) # fija la semilla del generador de nmeros 
            # aleatorios para obtener resultados reproducibles
p_tr<-0.7 #porcentaje de datos usados para el entrenamiento
N_datos<-dim(datos_advertising)[1] # nmero de registros del conjunto de datos
n_tr<-round(N_datos*p_tr) # este ser el tamao del conjunto de entrenamiento
ix_tr<-sample(N_datos,n_tr,replace = FALSE) # genera una muestra de ndices
datos_tr<-datos_advertising[ix_tr,] # conjunto de entrenamiento
datos_vl<-datos_advertising[-ix_tr,] # conjunto de validacin validacin

El siguiente cdigo implementa el mtodo de los vecinos ms cercanos del paquete caret. Luego utiliza la funcin predict() para predecir los datos de entrenamiento y calcula el MSE:

library(caret) # contiene una implementacin del mtodo de los kNN
adv_knn<-knnreg(sales~TV+radio+newspaper,data=datos_tr,k=3) # evala el mtodo con k=3
y_tr_pred<-predict(adv_knn,datos_tr)
mse_tr<-mean((datos_tr$sales-y_tr_pred)^2) # calcula el mse de entrenamiento

Cul es el nmero de vecinos ptimo?

Primero creamos una funcin que nos permita obtener el MSE como una funcin del nmero de vecinos:

mse_k<-function(k,data_tr,data_vl,formula_mod){
  adv_knn<-knnreg(formula_mod,data=datos_tr,k=k) # entrenamiento del modelo
  y_tr_pred<-predict(adv_knn,datos_tr) # prediccin de los valores de entrenamiento
  mse_tr<-mean((datos_tr$sales-y_tr_pred)^2) # mse del entrenamiento
  y_vl_pred<-predict(adv_knn,datos_vl) # prediccin de los valores de prueba
  mse_vl<-mean((datos_vl$sales-y_vl_pred)^2) # mse de validacin
  return(list(mse_tr=mse_tr,mse_vl=mse_vl))
  }

Cmo se utiliza esta funcin? El siguiente cdigo muestra cmo hacerlo:

modelo<-formula("sales~TV+radio+newspaper")
mse_k(k=2,data_tr=data_tr,data_vl = data_vl,formula_mod = modelo)

Ahora, se aplica la funcin mse_k() a los valores de \(k\) entre 1 y el nmero de datos en el conjunto de entrenamiento:

MSE<-lapply(1:n_tr,mse_k,data_tr=data_tr,data_vl = data_vl,formula_mod = modelo)

El objeto MSE es una lista de listas: cada entrada de MSE es una lista con dos componentes. Para acceder a sus elementos de manera organizada pordemos utilizar la funcin [ y la funcin sapply() como se muestra:

mse_tr<-sapply(1:n_tr,function(x,y){`[[`(y,x)$mse_tr},y=MSE)
mse_vl<-sapply(1:n_tr,function(x,y){`[[`(y,x)$mse_vl},y=MSE)

Ahora ya pueden graficarse los MSE de entrenamiento y validacin en funcin del nmero de vecinos:

Se ejecuta con el metodo train de caret, el cual requiere los parametros trControl. En este caso se utiliza el method=cv indicando que se utilizar el Cross Validation, esto es, dividir los datos de entrenamiento en 10 partes y hacer 10 modelos cada uno excluyendo 1/10 parte de los datos. Con esto se logra mejorar la estimacin del error utilizando el 100% de los datos. En la funcin train, se realiza el modelo con todas las variables usando los datos datos_tr. Aunque en este caso no se requiere, se realiz una estandarizacin de las variables con preProcess=c(center, scale).

trcntrl = trainControl(method="cv", number=10)
caret_knn_fit = caret::train(sales~TV+radio+newspaper, data=datos_tr,
                      method = "knn", trControl = trcntrl,
                      preProcess=c("center", "scale"),
                      tuneLength = 10)
